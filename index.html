📖 JavaScript Reference

Key concepts and all the functions you need for the challenges

Key Concepts
Variable
A named container that stores a value. Use let to create one. You can change its value later.
let score = 0;
score = score + 10;  // now 10

let name = "Ada";
let fruits = ["apple", "banana"];
String
A piece of text, wrapped in quotes. Strings have built-in methods like .toLowerCase() and .split().
let greeting = "Hello World";
greeting.length        // → 11
greeting.toLowerCase() // → "hello world"
greeting.split(" ")    // → ["Hello", "World"]
Array
An ordered list of values, wrapped in square brackets []. Each item has a numbered position (starting at 0).
let colors = ["red", "green", "blue"];
colors[0]         // → "red"
colors.length     // → 3

// Loop through every item:
for (let c of colors) {
  console.log(c);
}
Object
A collection of named properties (key-value pairs), wrapped in curly braces {}. Great for grouping related data together.
let student = {
  name: "Alex",
  grade: 10,
  enrolled: true
};
student.name      // → "Alex"
student.grade     // → 10
student["name"]   // → "Alex" (bracket notation)
Function
A reusable block of code that does a specific job. You call (run) it by writing its name followed by parentheses (). You can pass values in the parentheses for it to work with.
// Calling a built-in function:
Math.log(10)      // → 2.302…
console.log("hi") // prints "hi"

// Defining your own function:
function add(a, b) {
  return a + b;
}
add(3, 5)         // → 8
Method
A method is just a function that's attached to a value. You call it with a dot then the name, then parentheses. Strings, arrays, and objects all come with useful built-in methods.
// String methods:
"hello".toUpperCase()     // → "HELLO"
"a b c".split(" ")        // → ["a", "b", "c"]

// Array methods:
[3, 1, 2].filter(n => n > 1)  // → [3, 2]

// The dot before the name means
// "call this method on that value"
Comment
A note in your code that the computer ignores. Comments start with // and are just there to help humans understand what the code does.
// This is a comment — it does nothing
let x = 5;  // you can put them at the end too

// Comments are great for explaining
// WHY your code does something
Strings
.toLowerCase()
Converts a string to all lowercase letters.
"Hello World".toLowerCase()
// → "hello world"
.replace(pattern, replacement)
Replaces parts of a string that match a pattern. Use /g at the end to replace all matches.
"a!b?c".replace(/[!?]/g, " ")
// → "a b c"

// Remove everything except letters,
// numbers, apostrophes, spaces, hyphens:
"Hi!! I'm fine.".replace(/[^a-z0-9'\s-]/g, " ")
// → "Hi   I'm fine "
.split(separator)
Splits a string into an array. Use /\s+/ to split on one or more spaces.
"hello world".split(" ")
// → ["hello", "world"]

"a  b   c".split(/\s+/)
// → ["a", "b", "c"]
Arrays
.filter(fn)
Returns a new array keeping only items where the function returns true.
[1, 2, 3, 4].filter(n => n > 2)
// → [3, 4]

["a", "hi", "b"].filter(w => w.length > 1)
// → ["hi"]
for...of loop
Loops through each item in an array.
let fruits = ["apple", "banana"];
for (let fruit of fruits) {
  console.log(fruit);
}
// prints: "apple", then "banana"
Objects
Reading & writing properties
Access object values with dot or bracket notation.
let obj = { count: 0, name: "test" };
obj.count          // → 0
obj.count++        // now obj.count is 1
obj.count = obj.count + 5  // now 6

// Bracket notation (for variable keys):
let key = "name";
obj[key]           // → "test"
|| 0 — default to zero
If a property doesn't exist yet, obj[key] returns undefined. Use || 0 to treat it as 0.
let counts = {};
counts["hello"]            // → undefined
counts["hello"] || 0       // → 0

// Counting pattern:
counts["hello"] = (counts["hello"] || 0) + 1;
// now counts["hello"] is 1
Object.entries(obj)
Returns an array of [key, value] pairs. Useful for looping through all properties.
let scores = { positive: 5, negative: 3 };
for (let [label, value] of Object.entries(scores)) {
  console.log(label, value);
}
// prints: "positive" 5, then "negative" 3
Math
Math.log(x)
Natural logarithm. Used in Naive Bayes to prevent very small numbers from underflowing.
Math.log(1)    // → 0
Math.log(0.5)  // → -0.693...
Math.log(2)    // → 0.693...

// Key property: log(a * b) = log(a) + log(b)
// So we ADD log-probabilities instead
// of multiplying tiny probabilities
-Infinity
The smallest possible number. Useful as a starting value when looking for the maximum.
let best = -Infinity;
if (5 > best) { best = 5; }   // best is now 5
if (3 > best) { }             // 3 < 5, no change
if (9 > best) { best = 9; }   // best is now 9
Comparisons & If Statements
if statement
Runs code only when a condition is true.
let score = 10;
if (score > 5) {
  console.log("high score!");
}
// prints: "high score!"

// Comparison operators:
// >  greater than
// <  less than
// >= greater or equal
// === exactly equal

Deep Dive

How Does This Actually Work?

You just trained a Naive Bayes classifier. But how is that different from the AI chatbots you may have used? Let's break it down.

📊

Your Model: Naive Bayes Classifier

A simple word-counting algorithm

1
Count words per label. During training it looks at every word in your labeled examples and counts how often each word appears in "supportive" vs. "harmful" posts.
2
Calculate probabilities. For a new sentence, it asks: given each word in this sentence, what's the probability it belongs to each category? It multiplies those probabilities together (that's the "naive" part — it assumes words are independent).
3
Pick the winner. Whichever label has the higher combined probability wins.
Example

If your training data has the word proud appearing 8 times in supportive posts and 0 times in harmful posts, then seeing "proud" in a new sentence pushes the model strongly toward supportive.

But it has no understanding of what "proud" means. It's just counting.

🔢 Counts words ⚡ Instant training 📏 Needs 20–100 examples 🚫 No context awareness 🎯 Two categories only
🧠

Large Language Models (LLMs)

ChatGPT, Claude, Gemini, etc.

1
Pre-trained on the internet. LLMs are trained on billions of pages of text — books, websites, forums, Wikipedia, code, social media. This takes weeks on thousands of specialized chips and costs millions of dollars.
2
Learn patterns, not just word counts. They use neural networks with billions of parameters to learn relationships between words, grammar, tone, sarcasm, and even world knowledge. They process words in context — the meaning of "sick" in "that's sick!" vs. "I'm feeling sick" is different, and an LLM can often tell.
3
Fine-tuned with human feedback. After pre-training, humans rate the model's outputs to teach it to be helpful, harmless, and honest. This is called RLHF (Reinforcement Learning from Human Feedback).
🧠 Understands context 🐢 Months to train 📚 Billions of examples 💰 Millions of $$$ 🗣️ Generates text 🎭 Can detect sarcasm (sometimes)

Side-by-Side Comparison

📊 Your Naive Bayes 🧠 Large Language Model
How it learns Counts word frequencies per label Learns deep patterns from billions of sentences
Training data 20–100 examples you labeled Billions of web pages, books, etc.
Training time Milliseconds, in your browser Weeks–months on thousands of GPUs
Understands context? ❌ No — treats each word independently ✅ Yes — considers word order, tone, meaning
Handles sarcasm? ❌ Very poorly ⚠️ Sometimes, but not reliably
Handles reclaimed language? ❌ Can't tell context of use ⚠️ Better, but still makes mistakes
Bias risk Directly reflects your labeling choices Reflects biases in internet-scale training data
Can you inspect why? ✅ Yes — you can see the word counts ❌ Mostly a black box
Cost Free, runs in a browser $10M–$100M+ to train from scratch

🔑 The Key Insight

Your tiny classifier and a massive LLM are more alike than you think. Both learn from data humans provided. Both inherit the biases in that data. The difference is scale — and the fact that with your small model, you can see exactly where the bias comes from.

When an LLM makes a biased decision, it's much harder to trace why — the bias is buried in billions of parameters trained on data nobody fully audited. That's what makes the problem so hard at scale.
